JdbcMsSql - Users' Manual - Usage

Using the Binaries of the Project

This JDBC driver for DB/2 is to be used like any other JDBC driver. It is recommended, that the static method Db2Driver.register() is used to register it with the DriverManager.

The support for executing SQL is based on a parser for SQL:2008 and is limited to the single-table query functionality needed for SIARD Suite 2.1.

A GUI for JdbcMsSql

JdbcMsSql is distributed as a collection of JAR files without a graphical user interface. However, the versatile free application SQuirreL SQL Client can be used to make use of it.

UDT-Handling

The Microsoft implementation of dmd.getTypeInfo() returns DISTINCT types, which should really be returned in dmd.getUDTs(). And dmd.getUDTs() returns an empty ResultSet. The wrapper only returns the predefined types in dmd.getTypeInfo() and returns âaliasâ types as DISTINCT, Table-types as STRUCT, and CLR user-defined types as JAVA_OBJECT in dmd.getUDTs().

However, SQL Server 2012 (up to 2016) does not make it possible to access the âattributesâ of either (Table or CLR) UDT. Only CLR-UDTs really are of interest for database archival (SIARD). All CLR-UDTs implement Parse() and ToString(). Therefore all CLR UDTs are inserted using a parsable character string and are to be retrieved as strings (varchar) by selecting <col>.ToString() as <col>. If just the value <col> is selected, the serialized âbinaryâ data is returned.
